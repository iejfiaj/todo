# 📝 TodoMaster

순수 HTML / CSS / JavaScript로 만든 가벼운 할 일 관리 웹 앱입니다. 외부 라이브러리, CDN, 빌드 도구 없이 브라우저에서 바로 실행됩니다.

## 실행 방법

1. 이 폴더의 `index.html` 파일을 더블클릭하거나 브라우저로 열기
2. 끝. (서버나 설치 과정 없음)

## 파일 구조

```
TodoMaster/
├── index.html          # UI 마크업
├── style.css           # 스타일 (모바일 반응형 포함)
├── app.js              # 모든 동작 로직
├── manifest.json       # PWA 매니페스트
├── service-worker.js   # 오프라인 캐시
├── icon.svg            # 앱 아이콘 (벡터)
├── icon-192.png        # 192x192 PNG
├── icon-512.png        # 512x512 PNG
├── start-app.command   # 로컬 서버 + 앱 창 실행 스크립트
└── README.md           # 이 문서
```

## 데스크탑 앱처럼 설치하기 (PWA)

Chrome으로 두 가지 경로가 있어요. **B 방법(설치형 PWA)**가 가장 "진짜 앱" 같습니다.

### A. 빠른 방법 — 바로가기 만들기 (서버 불필요)

1. Chrome에서 `index.html` 열기 (`open -a "Google Chrome" index.html`)
2. Chrome 메뉴(⋮) → **도구 더보기 → 바로가기 만들기...**
3. **"창으로 열기"** 체크 → **만들기**
4. 데스크탑/Launchpad에 To-do 아이콘이 생기고, 클릭하면 주소창 없는 전용 창으로 열립니다

### B. 정식 PWA 설치 (오프라인 지원, 서버 필요)

1. **`start-app.command`** 더블클릭
   - 처음 실행 시 macOS가 막을 수 있어요. **시스템 설정 → 개인정보 보호 및 보안**에서 "확인 없이 열기" 누르면 됩니다.
   - 로컬에서 9876 포트로 서버가 뜨고 Chrome 앱 창이 자동으로 열립니다.
2. 열린 창의 메뉴(⋮) → **To-do 설치** 또는 **앱 설치**
3. 설치 완료 → Applications, Launchpad, Dock 어디에서든 진짜 앱처럼 실행 가능
4. 설치 후에는 서비스 워커가 모든 자원을 캐시하므로 **서버를 끄고도** 앱 아이콘만으로 동작합니다.

> 서버를 명시적으로 끄려면: `lsof -ti:9876 | xargs kill`

## 주요 기능

- **할 일 추가**: 입력창에 내용을 쓰고 카테고리 선택 후 `추가` 버튼 또는 Enter
- **카테고리**: 학교 공부 (파랑) / 개인 공부 (초록) / 업무 (주황) / 개인 일정 (보라)
- **달력**: 하단 달력에서 날짜 클릭 → 그 날짜의 할 일만 표시. 같은 날짜 다시 클릭 또는 "전체 날짜 보기"로 해제
- **하트 표시**: 그 날짜의 할 일을 모두 완료하면 달력에 ♥ 표시
- **완료 체크**: 체크박스 클릭 → 취소선 표시 + 진행률 즉시 갱신
- **수정**: ✏️ 버튼 → 인라인 편집 모드 (Enter 저장 / Esc 취소)
- **삭제**: 🗑️ 버튼 → 확인 후 삭제
- **카테고리 필터**: 상단 탭으로 전환. 진행률은 필터된 항목 기준으로 계산
- **자동 저장**: 모든 변경 사항이 `localStorage`에 즉시 저장. 새로고침해도 유지

## 검증 시나리오

브라우저에서 `index.html`을 열고 아래 순서대로 확인하세요.

### a. 항목 5개 추가 (카테고리 분산)
- "수학 과제" — 학교 공부
- "영단어 외우기" — 개인 공부
- "주간 보고서" — 업무
- "독서 30분" — 개인 공부
- "팀 미팅 준비" — 업무

→ 리스트에 5개가 카테고리 색 배지와 함께 표시되고 진행률이 `0/5 완료 (0%)`인지 확인

### b. 일부 완료 체크
- "수학 과제"와 "영단어 외우기"를 체크

→ 두 항목에 취소선이 그어지고 진행률이 `2/5 완료 (40%)`로 바뀌는지 확인

### c. 새로고침 → 데이터 유지
- 브라우저 새로고침 (Cmd+R / F5)

→ 5개 항목과 체크 상태, 카테고리가 모두 그대로 유지되는지 확인

### d. 카테고리 필터 동작
- `학교 공부` 탭 클릭 → "수학 과제" 1개만 표시되고 진행률 `1/1 완료 (100%)`
- `업무` 탭 클릭 → 2개 표시, 진행률 `0/2 완료 (0%)`
- 빈 카테고리로 전환 시 안내 문구 표시되는지 확인
- `전체` 탭 클릭 → 다시 5개 모두 표시

### e. 수정 / 삭제 동작
- 한 항목의 ✏️ 클릭 → input + select + 저장/취소 버튼으로 변경되는지 확인
- 내용/카테고리 변경 후 `저장` → 변경 반영
- 다른 항목 ✏️ 클릭 시 이전 편집은 자동 취소되는지 확인
- 빈 내용으로 저장 시 경고가 뜨고 저장되지 않는지 확인
- 🗑️ 클릭 → 확인 다이얼로그 → 확인 시 삭제

### f. 완료 항목 일괄 삭제
- 완료 항목이 있을 때 하단 우측 `완료 항목 모두 삭제` 버튼이 보이는지 확인
- 클릭 → 확인 후 완료된 항목만 모두 제거되는지 확인
- 완료 항목이 0개가 되면 버튼이 사라지는지 확인

## 데이터 저장 위치

브라우저의 `localStorage`에 다음 키로 저장됩니다.

- 키: `todoApp.tasks`
- 값: tasks 배열의 JSON 문자열

> 다른 브라우저나 시크릿 모드에서는 데이터가 공유되지 않습니다.
